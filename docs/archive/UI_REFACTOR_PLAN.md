# UI/UX 架构重构计划（Phase 1）

## 当前问题分析

现有项目采用 Activity 功能拆分模式：

```
MainActivity
FavoritesActivity
SettingsActivity
LockActivity
```

虽然逻辑清晰，但存在以下问题：

### 问题1：功能导向而非任务导向

当前结构：

```
首页
↓
收藏
↓
设置
↓
应用锁
```

属于开发者视角。

用户并不关心 Activity。用户只关心：

- 浏览内容
- 搜索内容
- 收藏内容
- 管理内容

---

### 问题2：页面层级割裂

收藏页：独立 Activity

设置页：独立 Activity

导致：

```
MainActivity → FavoritesActivity → 返回 MainActivity
MainActivity → SettingsActivity → 返回 MainActivity
```

产生频繁页面跳转，缺少连续性。

---

### 问题3：首页承担职责过重

当前 MainActivity：

- 首页
- 网站入口
- WebView浏览器
- 视频检测
- 播放入口

多个角色混杂，导致界面层级混乱。

---

### 问题4：底部导航无明确信息架构

当前：

```
首页 | 搜索（未实现） | 收藏 | 设置
```

存在问题：

- 搜索是核心功能却不可用
- 收藏和设置权重过高
- 首页定义模糊

---

## 用户任务流重构

重新梳理用户真实行为：

```
启动应用
↓
发现内容
↓
浏览内容
↓
观看内容
↓
收藏内容
↓
再次访问
```

这是主流程。

---

### 一级任务优先级

| 任务 | 描述 | 权重 |
|------|------|------|
| 浏览 | 用户主要行为 | 最高 |
| 搜索 | 用户主动查找内容 | 第二 |
| 收藏 | 用户管理个人内容 | 第三 |
| 设置 | 低频行为 | 最低 |

---

## 新信息架构

### 一级导航

推荐：

```
浏览 | 搜索 | 收藏 | 我的
```

而不是：

```
首页 | 搜索 | 收藏 | 设置
```

---

## 页面结构规划

### 浏览页

应用首页，展示：

- 常用站点
- 最近访问
- 推荐入口

作用：内容发现中心。

---

### 搜索页

核心入口，采用 Material3 SearchBar，支持：

- 历史记录
- 联想建议
- 最近搜索

---

### 收藏页

统一内容管理，支持：

- 收藏夹
- 最近访问
- 历史记录

避免多个管理入口。

---

### 我的页面

替代设置页，包含：

- 隐私保护
- 应用锁
- 图标伪装
- 广告规则
- 外部播放器
- 关于

符合现代 App 习惯。

---

## 浏览器模式重构

### 当前

```
首页 → 点击网站 → WebView
```

切换感明显。

### 推荐

```
浏览页 → 点击网站 → 进入沉浸式浏览模式
```

浏览模式包含：

- 顶部地址栏
- WebView
- 视频按钮
- 收藏按钮

形成独立浏览空间。

---

## 收藏逻辑重构

### 当前

```
网页 → 收藏 → FavoritesActivity
```

### 推荐

```
网页 → 收藏 → 统一内容中心
```

收藏只是内容状态，不是独立业务。

---

## 设置逻辑重构

### 当前

```
设置页
↓
广告规则
↓
应用锁
↓
图标伪装
```

层级过深。

### 推荐

```
我的
├─ 隐私与安全
├─ 浏览体验
├─ 广告拦截
├─ 外部播放器
├─ 图标伪装
└─ 关于应用
```

符合 Material 3 信息架构。

---

## Material 3 页面优先级

### 一级页面

- 浏览
- 搜索
- 收藏
- 我的

### 二级页面

- 网站详情
- 浏览器模式
- 历史记录
- 广告规则

### 三级页面

- 应用锁
- 图标伪装
- 高级设置

---

## 第一阶段目标

不修改业务逻辑，仅完成：

1. 信息架构梳理
2. 页面职责重新划分
3. 导航体系重构
4. 用户任务流重构
5. Material3页面层级设计

完成后进入：

- Phase 2：组件体系重构
- Phase 3：视觉设计重构
- Phase 4：动画与交互重构
